# Internationalization (i18n)

BiodataCraft supports 7 languages using a custom lightweight i18n system.

## Supported Languages

| Code | Language | Script | Font |
|------|----------|--------|------|
| `en` | English | Latin | DM Sans |
| `hi` | Hindi | Devanagari | Noto Sans Devanagari |
| `mr` | Marathi | Devanagari | Noto Sans Devanagari |
| `gu` | Gujarati | Gujarati | Noto Sans Gujarati |
| `ta` | Tamil | Tamil | System default |
| `te` | Telugu | Telugu | System default |
| `bn` | Bengali | Bengali | System default |

## Architecture

```
┌──────────────────────┐
│   I18nProvider        │  ← Wraps entire app in layout.tsx
│   (React Context)     │
│                       │
│   State: locale       │  ← Persisted to localStorage
│   Setter: setLocale   │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  useI18n() hook       │  ← Used by any component
│                       │
│  Returns:             │
│  - locale             │
│  - setLocale(locale)  │
│  - t (translations)   │
└──────────────────────┘
```

### Key Files

| File | Purpose |
|------|---------|
| `src/lib/i18n/index.ts` | Context, `LOCALES` array, `getTranslations()`, `useI18n()` hook |
| `src/lib/i18n/translations/en.ts` | English translations (source of truth) |
| `src/lib/i18n/translations/hi.ts` | Hindi translations |
| `src/lib/i18n/translations/mr.ts` | Marathi translations |
| `src/lib/i18n/translations/gu.ts` | Gujarati translations |
| `src/lib/i18n/translations/ta.ts` | Tamil translations |
| `src/lib/i18n/translations/te.ts` | Telugu translations |
| `src/lib/i18n/translations/bn.ts` | Bengali translations |
| `src/components/providers/i18n-provider.tsx` | Provider component |
| `src/components/ui/language-switcher.tsx` | Language dropdown |

## Usage

### In Components

```typescript
import { useI18n } from "@/lib/i18n";

function MyComponent() {
  const { locale, t, setLocale } = useI18n();

  return (
    <div>
      <h1>{t.landing.hero.title}</h1>
      <p>{t.landing.hero.subtitle}</p>
      <p>Current language: {locale}</p>
    </div>
  );
}
```

### In Templates

Template utility functions accept an optional `locale` parameter:

```typescript
import { getPersonalFields, getLabel } from "./template-utils";

// Inside template component:
const fields = getPersonalFields(data, locale);
const nameLabel = getLabel("fullName", locale);
```

### Language Switcher

The `<LanguageSwitcher />` component renders a dropdown with all supported languages, showing each language's native name.

## Adding a New Language

### Step 1: Create Translation File

Create `src/lib/i18n/translations/xx.ts` (where `xx` is the ISO 639-1 code):

```typescript
import { DeepStringRecord } from "./en";

const xx: DeepStringRecord = {
  landing: {
    hero: {
      title: "Translated title",
      subtitle: "Translated subtitle",
      // ... all keys from en.ts
    },
    // ... all sections
  },
  form: {
    steps: {
      personal: "Translated step name",
      // ... all steps
    },
    fields: {
      fullName: "Translated field label",
      // ... all fields
    },
  },
  // ... all top-level sections from en.ts
};

export default xx;
```

Use `en.ts` as the reference — every key in the English file must have a corresponding key in the new translation.

### Step 2: Register the Language

In `src/lib/i18n/index.ts`:

```typescript
import xx from "./translations/xx";

export const LOCALES = [
  // ... existing locales
  { id: "xx" as const, label: "LanguageName", nativeLabel: "NativeLabel" },
];

// Update the Locale type (auto-derived from LOCALES)
// Update getTranslations() to include the new language
```

### Step 3: Add Font (if needed)

If the script requires a special font, add it in `src/app/layout.tsx`:

```typescript
import { Noto_Sans_NewScript } from "next/font/google";

const notoNewScript = Noto_Sans_NewScript({
  subsets: ["newscript"],
  variable: "--font-newscript",
  weight: ["400", "500", "600", "700"],
});

// Add to body className: ${notoNewScript.variable}
```

## Type Safety

The English translation file exports a `DeepStringRecord` type utility:

```typescript
type DeepStringRecord<T = Record<string, unknown>> = {
  [K in keyof T]: T[K] extends Record<string, unknown>
    ? DeepStringRecord<T[K]>
    : string;
};
```

This ensures all translations have the same structure as the English source without requiring literal string types (which would fail for translated text).

## Regional Landing Pages

For SEO, dedicated landing pages exist at `/hindi` and `/gujarati`. These are static marketing pages (not i18n variants of the app) with:

- Full native-language content
- SEO metadata in the target language
- Region-specific testimonials and FAQs
- JSON-LD structured data with `inLanguage` set
